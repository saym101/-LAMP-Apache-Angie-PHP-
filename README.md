# LAMP/LEMP Installer

![Version](https://img.shields.io/badge/version-4.0.1-blue)
![License](https://img.shields.io/badge/license-MIT-green)
![Platform](https://img.shields.io/badge/platform-Debian%2FUbuntu-orange)

Автоматизированный интерактивный скрипт для установки и базовой настройки веб-стека на Debian/Ubuntu:

- Angie + PHP-FPM 8.3;
- Apache + PHP-FPM 8.3;
- MariaDB с первичной безопасной настройкой root-пароля;
- тестовая страница и скрытый `phpinfo()`;
- выпуск SSL-сертификатов Certbot для домена или публичного IPv4;
- резервное копирование конфигурации перед сменой веб-сервера.

## Поддержка

Скрипт рассчитан на Debian и Ubuntu с `systemd`, `apt` и root-доступом.

Для PHP 8.3 используется:

- Debian: репозиторий `packages.sury.org`;
- Ubuntu: штатные репозитории, если пакет доступен, иначе `ppa:ondrej/php`.

Для IP-сертификатов Let's Encrypt нужен Certbot 5.3+ с поддержкой `--ip-address` и `--preferred-profile shortlived`. Если системный пакет Certbot старее, пункт IP-сертификата завершится понятной ошибкой и попросит обновить Certbot.

## Быстрый старт

```bash
sudo bash lamp.sh
```

После запуска выберите в меню:

1. `Установить Angie + PHP 8.3`;
2. `Установить Apache + PHP 8.3`;
3. `Установить MariaDB`;
4. `Настроить SSL (Certbot)`.

## Важные файлы

- Лог запуска: `lamp_YYYY-MM-DD.log` в каталоге, из которого запущен скрипт.
- Бэкапы конфигурации: `/backups/web-lamp/`.
- Состояние скрипта: `/var/lib/lamp-setup/`.
- Пароль MariaDB root: `/root/mariadb_credentials.txt`.
- Подсказки cron для Certbot: `/root/certbot_cron.txt`.
- Корень сайта: `/var/www/html`.

## Безопасность

Скрипт создаёт тестовую страницу и файл `phpinfo()`. После проверки удалите их через пункт меню `Удалить тестовые файлы`, потому что `phpinfo()` раскрывает данные о системе и PHP-модулях.

Перед заменой Apache на Angie или Angie на Apache скрипт предлагает создать архив текущей конфигурации. Пользовательские сайты и нестандартные виртуальные хосты всё равно стоит дополнительно сохранить вручную.

## Что делает скрипт

- Проверяет поддерживаемую ОС и устанавливает базовые зависимости.
- Подключает нужные репозитории для PHP/Angie.
- Настраивает PHP-FPM socket для `www-data`.
- Создаёт базовый virtual host для выбранного веб-сервера.
- Настраивает безопасные заголовки и отключает лишнюю выдачу версии PHP.
- При активном UFW предлагает открыть порты `80/tcp` и `443/tcp`.
- Для MariaDB генерирует случайный root-пароль и сохраняет его в файл с правами `600`.

## Ограничения

- Скрипт не переносит существующие сайты и сложные virtual host конфигурации.
- SSL по домену требует корректной DNS A/AAAA-записи на сервер.
- SSL по IP требует публичный IPv4, открытый порт 80 и свежий Certbot.
- Автообновление Certbot не записывается в crontab автоматически: готовая строка сохраняется и выводится на экран.
